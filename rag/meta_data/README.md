# Meta Data Project

## Purpose

The `meta_data` folder is designed to enrich parsed data generated by our parser for specific projects with **project-related metadata**. This enriched data is stored in a dedicated MongoDB collection associated with the respective project for future retrieval and analysis.

### Key Features

- Enrichment of parsed objects with metadata tailored to specific projects.
- Storage of enriched objects in MongoDB for efficient access.
- Utilization of recurring terms frequently referenced within the project to ensure metadata relevance.

## Implementation

The following classes have been implemented to achieve the project's objectives:

### 1. **MetaDataProjectExpander**

This class handles:

- Gathering project-specific details.
- Processing all parsed elements generated by the parser.
- Enriching each object with metadata.
- Inserting the enriched objects into a dedicated MongoDB collection.

### 2. **MetaDataQueryExpander**

This class is responsible for:

- Accepting a user query as input.
- Extracting relevant metadata from the query to identify key information.

### 3. **MetaDataRetriever**

This class focuses on:

- Accepting metadata derived from a user query.
- Finding the best-matching elements in the MongoDB collection that align with the provided query metadata.
- Returning the most relevant results for the given query.

### Creating a New Metadata Extractor for a Project

To integrate the **Retrieval-Augmented Generation (RAG)** feature for contextual enrichment in a new project within **GENIE/TAG**, follow these steps:

1. **Run the NLP Term Extraction Script**

   - Execute the following script on the final **JSON/NDJSON** dataset generated by **DPR** for the target project: `rag/meta_data/nlp/tests/noun_prompt_lab_test.py`.
   - This script analyzes the project’s codebase and prints the **top 100 most frequently used terms/nouns**, which serve as the foundation for metadata extraction.

2. **Create a Project-Specific Metadata Extractor**

   - Using the extracted terms, implement a new metadata extractor class for the project based on `MetaDataExtractorBase`.

3. **Integrate the New Extractor Class**

   - Modify the following files to incorporate the new extractor class:
     - `rag/meta_data/helpers/meta_data_project_expander.py`
     - `rag/meta_data/helpers/meta_data_query_expander.py`
     - `rag/meta_data/helpers/metadata_extractor/__init__.py`

By following these steps, each project benefits from tailored metadata enrichment, enhancing the **efficiency and accuracy** of contextual data retrieval and project-specific analysis.
